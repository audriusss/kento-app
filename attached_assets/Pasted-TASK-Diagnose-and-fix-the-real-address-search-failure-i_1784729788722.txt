INVESTIGATE THESE LIKELY BUGS

1. onError immediately sets isNavigating = false, causing NavigationScreen disposal and engine onDestroy.
2. DisposableEffect calls onDestroy during recomposition.
3. NavigationController or engine is recreated on every recomposition.
4. remember is missing around NavigationController / GoogleNavigationEngine.
5. engine lifecycle is tied to NavigationScreen instead of Activity/ViewModel.
6. stopNavigation is called from a BackHandler or UI state transition.
7. startNavigation fails because navigator is not initialized yet.
8. the Geocoder callback returns after the engine has already been destroyed.
9. wrong Log tag is being filtered.
10. MockNavigationEngine fallback or engine switch is occurring unexpectedly.

REQUIRED ARCHITECTURE FIX

- GoogleNavigationEngine must not be recreated on every Compose recomposition.
- Keep NavigationController / engine in a ViewModel or remember with stable application context.
- onDestroy must run only when Activity/ViewModel is genuinely destroyed.
- stopNavigation must not call onDestroy.
- A failed address search should update error state, not destroy the navigation engine.
- Prevent stale async geocoder callbacks from acting after a newer request, but do not cancel the engine itself.
